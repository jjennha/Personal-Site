WITWAY

WITWAY is a service for users who want an easy way to plan a trip in any US location. Travelers enter their preferences in a form, and a trip is generated with venue and directional information. Travelers can also be added to groups making trip planning for multiple people easier.

I built this Android Mobile application with a team over the course of a semester. My main contribution was in the form-to-trip generation, as well as the routing feature on the app. I developed WITWAY's API, client side connections, and learned how to work with multiple different APIs during the course of completing these features.

Technologies used:

Springboot, Volley, YelpFusion, Google Maps SDK

Data Request System

I developed a web application with another intern for the Buildertrend Data Science Team to manage and archive company data request submissions. The previous system required requesters to download and edit a Word Document form and emailing the request to the Data Science Team. It was a lot of manual work and requests were sometimes missed due to processes being done in multiple locations. My co-worker and I were able to create a single web application that features a web form for the company employees to submit requests, a notification system that alerts the data science team by email when a request is submitted, a queue where requests are managed, and an archive of completed requests.

Technologies used:

C#, KnockoutJS, ASP.NET MVC, Entity Framework, SQL

ScreenToScript

I collaborated with a team of Buildertrend interns to create Google Chrome extension plugin for Katalon Recorder that translates and exports user actions on webpages to JavaScript Puppeteer files. This tool was created to assist Buildertrend's Tech Service Managers and Quality Assurance Engineers to recreate an errors that clients might be having with Buildertrend's products.

Technologies used:

JavaScript, Puppeteer, Node.js
